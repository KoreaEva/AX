## 실습 전 — AI의 한계 먼저 알기

박준혁은 기술자입니다. 도구를 쓰기 전에 한계부터 파악합니다.  
클로드를 자료 조사에 쓰기 전에 반드시 알아야 할 두 가지가 있습니다.

### ⚠️ 한계 1: 환각 (Hallucination)

AI가 사실이 아닌 정보를 마치 사실인 것처럼 생성하는 현상입니다.  
**클로드를 포함한 모든 AI는 환각을 일으킬 수 있습니다.**

| 환각 가능성 낮음 ✅ | 환각 가능성 높음 ⚠️ |
|-------------------|-------------------|
| "일반적으로", "보통" 같은 표현 사용 | "2024년 3월 15일" 같은 구체적인 날짜 |
| 잘 알려진 역사적 사실 | 최근 사건, 통계 수치 |
| 출처를 명시한 정보 | 출처 없이 단정적인 답변 |
| "확인이 필요합니다" 같은 유보적 표현 | "분명히", "확실히" 같은 단정적인 표현 |

> 박준혁의 교훈: "AI가 자신 있게 말해도 틀릴 수 있다. 네트워크 장비 스펙처럼 수치가 중요한 정보는 반드시 직접 확인해야 한다."

---

### ⚠️ 한계 2: 지식 컷오프 (Knowledge Cutoff)

AI는 특정 시점까지의 데이터로만 학습했기 때문에, 그 이후의 정보는 알지 못합니다.

**박준혁에게 실제로 문제가 되는 상황:**

- 최신 네트워크 보안 취약점 CVE 정보 → 컷오프 이후 발표된 건 모름
- 새로 출시된 방화벽·SD-WAN 장비 스펙 → 누락 가능성 큼
- 최근 개정된 정보보호 관련 법규 → 과거 버전을 안내할 위험
- 경쟁사 네트워크 인프라 현황 → 이미 수정된 최신 정보가 아닐 수 있음

> **핵심 원칙:** 웹 검색 없이 클로드에게 최신 기술 정보를 물어보는 것은  
> 사실상 거짓말을 해달라는 것과 같습니다.

## 실습 2 — 웹 검색으로 최신 보안 정보 가져오기

> **배우는 것:** 웹 검색 기능을 켜면 지식 컷오프 한계를 넘어 최신 정보를 얻을 수 있습니다.


## 실습 3 — 연구 모드로 네트워크 보안 솔루션 비교 보고서 만들기

> **배우는 것:** 연구 모드는 여러 각도에서 자료를 수집하고, 교차 검증하며,  
> 종합 리포트를 자동으로 작성합니다. 최팀장이 요청한 보고서를 만들어봅시다.

### 연구 모드 vs 일반 웹 검색 비교

| 항목 | 일반 웹 검색 | 연구 모드 |
|------|------------|---------|
| 검색 횟수 | 1~2회 | 5~10회 이상 |
| 소요 시간 | 빠름 (수초) | 느림 (10~20분) |
| 결과 깊이 | 단편적 정보 | 종합 분석 리포트 |
| 출처 교차검증 | 제한적 | 자동 비교·검증 |
| 적합한 상황 | 빠른 단순 조회 | 복잡한 의사결정 자료 |

> 박준혁의 판단: "최팀장 보고서는 벤더 비교와 비용 분석까지 들어가야 하니까 연구 모드가 맞다."

---

### 연구 모드 켜는 방법

```
새 채팅 열기 → 입력창 [+] 버튼 → [연구] 클릭 → 아이콘이 활성화되면 준비 완료
```

> **TIP:** 연구 모드에서는 오퍼스(Opus) 모델 + 적응형 사고 기능을 함께 켜면  
> 복잡한 기술 분석에서 더 정교한 결과를 얻을 수 있습니다.

---

### 실습 3 — 연구 모드 프롬프트 (직접 입력해보세요)

> **연구 모드 ON 상태에서 입력**

```
다음 조건으로 네트워크 보안 솔루션 검토 보고서를 작성해줘.

주제: KT 내부망 보안 강화를 위한 네트워크 보안 솔루션 도입 검토

포함 내용:
1. 2025년 주요 네트워크 보안 위협 트렌드
2. 차세대 방화벽(NGFW) 주요 벤더 4개사 비교 (기능, 성능, 가격대)
3. SD-WAN 도입 현황 및 보안 기능 비교
4. 국내 기업 도입 사례 및 효과
5. 예상 도입 비용 범위와 ROI 추정

분량: A4 4페이지 수준
형식: 마크다운으로 표와 불릿 포인트 활용
```

---

### 연구 진행 중 확인 포인트

연구 모드가 시작되면 오른쪽 패널(아티팩트)에서 진행 상황을 볼 수 있습니다.

- [ ] 연구 계획이 자동으로 작성되었나요?
- [ ] 몇 개의 출처가 수집되고 있나요?
- [ ] 내가 요청한 5가지 항목이 모두 포함되어 있나요?

> **주의:** 연구 모드는 완료까지 시간이 걸립니다.  
> 잘못된 방향으로 진행되는 것을 발견하면 과감히 중지하고 수정 지시를 내리세요.

---

### 결과물 품질 검증 체크리스트 (박준혁의 엔지니어 기준)

연구가 완료되면 아래 항목을 직접 확인합니다.

**정확성 검증:**
- [ ] 벤더사 제품명과 모델명이 실제 존재하는 제품인가?
- [ ] 성능 수치(처리 속도, 처리량 등)가 공식 데이터시트와 일치하는가?
- [ ] 가격 범위가 현실적인가? (너무 낮거나 높지 않은가?)
- [ ] 인용된 사례 기업이 실제로 해당 솔루션을 도입했는가?

**출처 신뢰성 검증:**
- [ ] 공식 제조사 문서, 공인된 리서치 기관 자료인가?
- [ ] 발행 연도가 2024~2025년인가?
- [ ] 국내 환경(규제, 법령)을 반영하고 있는가?

**누락 항목 확인:**
- [ ] 요청한 5가지 항목이 모두 포함되어 있는가?
- [ ] 보안 인증 요건(CC인증, ISMS 등)이 언급되어 있는가?

## 실습 4 — 불완전한 보고서 보완하기

> **배우는 것:** 연구 모드 결과물은 완성본이 아닙니다.  
> 추가 질문과 수정 지시로 품질을 높이는 방법을 익힙니다.

### 스토리

박준혁이 연구 모드 결과를 받았습니다. 전체적으로 괜찮지만...  
*"Cisco 비교 데이터가 너무 일반적이야. 구체적인 수치가 필요한데."*  
*"국내 KT 같은 통신사 환경에 맞는 내용이 부족해."*

---

### 보완 프롬프트 예시 (직접 입력해보세요)

**부족한 부분 구체화 요청:**

```
앞서 작성한 보고서에서 Cisco Firepower와 Palo Alto Networks 비교 부분을
더 구체적으로 보완해줘.
특히 처리량(Throughput), 동시 세션 수, 관리 편의성 항목을
실제 데이터시트 기준으로 표로 만들어줘.
```

**국내 환경 반영 요청:**

```
국내 통신사 및 대기업 환경에서 SD-WAN 도입 사례를
웹 검색으로 추가로 찾아서 보완해줘.
ISMS-P 인증 요건과의 연계성도 추가해줘.
```

**최팀장 보고 형식으로 정리:**

```
위 내용을 임원 보고용으로 다시 정리해줘.
- 분량: 1페이지 이내 요약본
- 형식: 핵심 결론 3줄 + 추천 솔루션 + 예상 비용
- 말투: 격식체, 간결하게
```

---

### 실습 포인트

- [ ] 연구 결과에서 부족한 부분을 직접 찾아보고 추가 질문을 작성해본다
- [ ] 같은 주제를 일반 웹 검색과 연구 모드로 각각 조사해서 품질을 비교한다
- [ ] 최종 보고서에서 출처를 클릭해 실제 원본 페이지를 확인해본다

## 박준혁의 자료 조사 원칙 (실습 후 정리)

> 기술자는 도구의 한계를 알고 써야 한다.

1. **웹 검색은 필수다** — 최신 기술 정보는 반드시 웹 검색 ON 상태에서 조사한다
2. **출처는 직접 확인한다** — 공식 제조사 문서, 공인 기관 자료만 최종 보고서에 인용한다
3. **수치는 교차 검증한다** — 성능 수치, 가격, CVE 정보는 원본 데이터시트와 반드시 대조한다
4. **복잡한 조사는 연구 모드다** — 단순 검색과 심층 분석을 상황에 맞게 골라 쓴다
5. **AI는 초안이다** — 최종 보고서는 내가 검토하고 책임진다
