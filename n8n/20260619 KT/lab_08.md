## 오늘의 배경

> 최동현 팀장은 매달 반복되는 업무에 지쳐 있습니다.
>
> - 매달 비슷한 형식의 **월간 예산 집행 보고서** 작성
> - 분기마다 반복되는 **계약 검토 체크리스트** 정리
> - 부서별로 올라오는 **비품 구매 요청서** 검토 및 회신
> - 매번 처음부터 다시 설명해야 하는 **총무팀 작성 기준과 용어**
>
> 이수아가 말했습니다: *"팀장님, 그 작업들 프로젝트로 한 번만 세팅해두면 매번 처음부터 설명 안 해도 돼요."*
>
> 최동현의 속마음: *"그게 진짜야? 한번 해봐야겠네."*

---

## 프로젝트란 무엇인가 — 최동현식 비유

| 구분 | 일반 대화 | 프로젝트 |
|------|----------|---------|
| 비유 | 처음 만난 직원에게 매번 업무 설명 | 오래 함께 일한 전담 보좌관 |
| 대화 지속성 | 한 세션만, 다음 번엔 처음부터 | 여러 대화가 연결되어 맥락 유지 |
| 파일 참고 | 매번 새로 첨부해야 함 | 한 번 올리면 계속 참고 |
| 맞춤 지침 | 전역 설정만 가능 | 프로젝트별로 다른 기준 설정 가능 |
| 대화 정리 | 시간순 나열, 찾기 어려움 | 프로젝트별 그룹화, 정리 가능 |
| 적합한 용도 | 단발성 질문 | 반복 업무, 장기 프로젝트 |

> 최동현의 판단: *"총무팀 업무는 매달, 매분기 반복되는 패턴이다. 프로젝트가 딱 맞다."*

## 실습 1 — 총무팀 전용 프로젝트 만들기

> **배우는 것:** 프로젝트를 생성하고, 총무팀 업무에 맞는 이름과 설명을 설정합니다.

### 스토리

최동현 팀장은 총무팀의 반복 업무를 체계화하기 위해 첫 프로젝트를 만들기로 했습니다.

---

### 프로젝트 만드는 방법

```
1. 클로드 화면 왼쪽 사이드바에서 [Projects] 클릭
2. 오른쪽 상단 [New project] 클릭
3. 프로젝트 이름과 설명 입력
4. [프로젝트 만들기] 클릭
```

---

### 최동현이 입력할 프로젝트 정보

```
프로젝트 이름: KT 총무팀 반복 업무 관리

설명: 월간 예산 보고서, 계약 검토, 비품 구매 요청 등 총무팀 반복 업무 전용 공간
```

---

### 실습 포인트

- [ ] 프로젝트 화면을 열고 [New project] 버튼을 찾는다
- [ ] 위 프로젝트 이름과 설명을 입력하고 생성한다
- [ ] 생성된 프로젝트가 왼쪽 사이드바에 나타나는지 확인한다

> **TIP:** 프로젝트는 무료 요금제에서도 사용할 수 있습니다.

---

## 실습 2 — 총무팀 맞춤 지침 설정하기

> **배우는 것:** 프로젝트 지침을 설정하면 매번 같은 설명을 반복하지 않아도 됩니다.  
> 지침도 컨텍스트입니다. 01.4에서 배운 프롬프트 원칙을 그대로 적용합니다.

### 스토리

최동현 팀장은 20년 경력 동안 총무팀만의 작성 기준이 있습니다.  
보고서 형식, 용어 정의, 금액 표기 방식... 매번 클로드에게 다시 설명하기가 귀찮았는데,  
지침에 한 번만 써두면 이 프로젝트의 모든 대화에 자동으로 적용됩니다.

---

### 지침 설정하는 방법

```
1. 생성된 프로젝트 화면에서 [지침] 섹션의 [+] 버튼 클릭
2. 지침 내용 입력
3. [지침 저장] 클릭
※ 지침은 언제든지 수정 가능합니다
```

---

### 최동현이 입력할 지침 예시

아래 내용을 복사해서 지침 입력창에 붙여넣으세요.

```
# KT 총무팀 업무 지침

## 기본 정보
- 소속: KT 총무팀
- 담당: 예산 집행, 계약 관리, 비품·자산 관리, 시설 운영
- 보고 대상: 경영지원본부장 (월간), 대표이사 (분기)

## 문서 작성 기준
- 모든 금액은 원(₩) 단위로 표기, 1,000원 단위 콤마 구분
- 예산 초과율은 소수점 첫째 자리까지 표기 (예: 3.2%)
- 날짜 형식: YYYY년 MM월 DD일
- 부서명은 공식 명칭 사용 (예: 네트워크운용팀, 인사팀, 법무팀)

## 보고서 표준 구조
월간 보고서는 반드시 아래 순서로 작성:
1. 핵심 요약 (3줄 이내)
2. 예산 집행 현황 (표)
3. 주요 계약 현황
4. 이슈 및 조치 사항
5. 차월 계획

## 어조 및 스타일
- 격식체 사용 (합니다, 입니다)
- 수동적 표현 지양, 능동적 서술 (예: "집행되었음" → "집행함")
- 수치 근거 없는 서술 금지
- 불확실한 내용은 반드시 "확인 필요" 또는 "추정치" 명시

## 금지 사항
- 실제 데이터 없이 수치를 임의로 생성하지 말 것
- 계약 관련 법적 판단은 반드시 "법무팀 검토 필요" 문구 추가
- 개인정보(직원 이름, 연락처) 포함 금지
```

---

### 실습 포인트

- [ ] 위 지침을 그대로 복사해서 입력해본다
- [ ] 지침을 저장한 후 "이 프로젝트의 지침을 요약해줘"라고 물어본다
- [ ] 내 실제 업무에 맞게 수정할 항목이 있으면 찾아서 메모한다

### 내가 수정하고 싶은 지침 항목

```
수정할 항목:
1. 
2. 
3. 

추가하고 싶은 내용:
```

> **프로젝트 지침 vs 전역 설정 우선순위:**  
> 프로젝트 지침이 설정의 [일반] 맞춤 설정보다 우선 적용됩니다.

## 실습 3 — 참고 파일 올리기

> **배우는 것:** 자료를 프로젝트에 한 번만 올리면 이후 모든 대화에서 자동으로 참고합니다.  
> 클로드가 실제 자료를 보고 생성한 결과는 환각 가능성이 낮습니다.

### 스토리

최동현 팀장은 총무팀 업무에 필요한 참고 자료들이 있습니다.  
매번 첨부하는 게 번거로웠는데, 프로젝트에 한 번만 올리면 해결됩니다.

---

### 파일 업로드 방법

```
프로젝트 화면에서 [파일] 섹션의 [+] 클릭
→ [기기에서 업로드] 선택
→ 파일 선택 후 업로드
※ PDF, Word, 텍스트, 이미지 등 다양한 형식 지원
※ GitHub, Google Drive 연동도 가능 (04장에서 다룹니다)
```

---

### 최동현이 올릴 파일 목록 (예시)

실제 업무에서 아래와 같은 파일을 올려두면 유용합니다.

| 파일 종류 | 활용 목적 |
|----------|----------|
| 월간 예산 집행 양식 (Excel/PDF) | 매달 보고서 작성 시 형식 기준 |
| 표준 계약서 템플릿 | 계약 검토 시 기준 문서 |
| 비품 구매 요청 양식 | 부서별 요청 검토 기준 |
| KT 내부 규정 (총무 관련) | 규정 기반 답변 생성 |
| 전년도 예산 집행 보고서 | 올해 보고서 작성 시 참고 |

### 실습 — 파일 업로드 후 확인 (직접 입력해보세요)

파일을 올린 후 아래 프롬프트를 입력해서 클로드가 파일을 제대로 읽었는지 확인합니다.

```
내가 이 프로젝트에 업로드한 파일 목록을 알려줘.

업로드된 파일은 총 N개입니다.

2. [파일명] — [내용 요약]
...
어떤 파일부터 작업을 시작할까요?
```

- [ ] 업로드 후 목록 확인 프롬프트를 입력해서 파일이 잘 올라갔는지 확인한다
- [ ] 클로드가 파일 내용을 정확히 파악하고 있는지 확인한다

## 실습 4 — 실제 자료 기반으로 업무 처리하기

> **배우는 것:** 지침과 파일이 준비되면 짧은 프롬프트만으로도 일관된 결과물을 얻을 수 있습니다.

### 스토리

프로젝트 세팅을 마친 최동현 팀장. 이제 실제 업무를 시켜봅니다.  
세팅 전에는 길게 설명해야 했지만, 이제는 한 줄로도 충분합니다.

---

### 업무 A: 월간 예산 집행 보고서 초안 작성

**세팅 전 (매번 이렇게 써야 했다):**

```
너는 KT 총무팀 담당자야. 격식체로, 금액은 원 단위로,
핵심 요약 → 예산 현황 표 → 주요 계약 → 이슈 → 차월 계획 순서로
6월 예산 집행 보고서 초안 써줘.
총 예산 5억, 집행액 3억 2천만원, 잔액 1억 8천만원이야.
```

**세팅 후 (지침이 있으니 이것만으로 충분):**

```
6월 예산 집행 보고서 초안 작성해줘.
총 예산 5억, 집행액 3억 2천만원, 잔액 1억 8천만원.
```

> 지침에 이미 형식, 어조, 구조가 모두 설정되어 있습니다.

---

### 직접 입력해보세요 — 업무 A

```
6월 예산 집행 보고서 초안 작성해줘.
총 예산 5억, 집행액 3억 2천만원, 잔액 1억 8천만원.
주요 집행 내역:
- 시설 유지보수: 8천만원
- 사무용품 및 비품: 2천5백만원
- 외부 용역비: 1억 5천만원
- 기타 운영비: 6천5백만원
이슈: 냉방 장비 노후화로 교체 검토 중
```

**결과 확인 포인트:**
- [ ] 지침에 설정한 형식(핵심요약→예산현황→이슈→차월계획)을 따랐나요?
- [ ] 금액이 ₩ 단위와 콤마로 올바르게 표기되었나요?
- [ ] 격식체(합니다, 입니다)를 사용했나요?
- [ ] 수치 근거 없는 서술이 없나요?

---

### 업무 B: 비품 구매 요청서 검토 (직접 입력해보세요)

```
네트워크운용팀에서 비품 구매 요청이 왔어.
아래 내용을 검토하고, 승인 여부와 의견을 작성해줘.

요청 내용:
- 품목: 사무용 의자 10개
- 단가: 28만원
- 합계: 280만원
- 요청 사유: 기존 의자 노후화 (사용 연수 8년)
- 요청 부서: 네트워크운용팀 박준혁
```

---

### 업무 C: 계약 검토 의견서 (직접 입력해보세요)

```
아래 계약 건에 대한 총무팀 검토 의견서 초안 작성해줘.

계약 개요:
- 계약 상대방: (주)클린서비스
- 계약 목적: 본사 사옥 청소 용역
- 계약 금액: 월 450만원 (연 5,400만원)
- 계약 기간: 2025년 8월 1일 ~ 2026년 7월 31일 (1년)
- 특이사항: 기존 업체 계약 만료에 따른 재계약 (3년 연속 동일 업체)
```

## 실습 5 — 프로젝트의 지속성 확인하기

> **배우는 것:** 프로젝트에서는 채팅창이 달라도 이전 작업 내용을 기억합니다.  
> 이것이 일반 대화와 프로젝트의 가장 큰 차이입니다.

### 스토리

실습 4에서 보고서를 작성한 최동현 팀장. 다음날 새 채팅을 열었습니다.  
일반 대화였다면 처음부터 다시 설명해야 하지만, 프로젝트라면 달라집니다.

---

### 테스트 — 새 채팅을 열고 입력해보세요

> **같은 프로젝트 안에서** 새 채팅(New chat)을 열고 아래 프롬프트 입력

```
이 프로젝트에서 지금까지 작성한 문서 목록을 알려줘.
```

**예상 응답:**
```
이 프로젝트에서 현재까지 작성된 문서는 N건입니다.

작성 완료:
1. 6월 예산 집행 보고서 초안
2. 비품 구매 요청 검토 의견
3. 계약 검토 의견서

다음 작업을 이어서 진행할까요?
```

> 채팅창이 달라도 같은 프로젝트 안의 내용은 서로 연결됩니다.

---

### 추가 테스트

```
지난번에 작성한 6월 예산 보고서에서 이슈 사항만 다시 알려줘.
```

- [ ] 새 채팅에서도 이전 내용을 정확히 참조하나요?
- [ ] 일반 대화에서 같은 질문을 하면 어떻게 응답하는지 비교해보세요

## 실습 6 — 프로젝트 메모리 관리하기

> **배우는 것:** 프로젝트 메모리는 모든 대화를 요약한 기록입니다.  
> 클로드가 같은 실수를 반복한다면 메모리에 잘못된 내용이 쌓인 것일 수 있습니다.

### 메모리 확인 방법

```
프로젝트 화면 → [메모리] 섹션 → [θ] 아이콘 클릭
→ 프로젝트 메모리 내용 확인 및 수정 가능
```

---

### 메모리 내용 한글로 확인하기 (직접 입력해보세요)

메모리는 영문으로 자동 작성됩니다. 내용 확인을 위해 번역을 요청합니다.

```
지금 이 프로젝트의 메모리 내용을 한글로 번역해줘.
```

---

### 메모리에 추가할 내용 요청 예시 (직접 입력해보세요)

```
메모리에 다음 내용을 추가해줘:
- 총무팀 예산 보고서는 매월 25일까지 경영지원본부장에게 제출한다
- 500만원 이상 계약은 반드시 법무팀 사전 검토가 필요하다
- 비품 구매는 분기별 예산 한도 내에서만 승인 가능하다
```

---

### 대화 정리하기 — 주제 벗어난 채팅 찾기 (직접 입력해보세요)

프로젝트가 쌓이면 관련 없는 대화도 생깁니다. 주기적으로 정리하면 좋습니다.

```
이 프로젝트 주제에서 어긋난 대화를 찾아봐.
```

**정리 방법:**
```
해당 채팅방의 [...] 메뉴 클릭
→ [프로젝트 변경]: 다른 프로젝트로 이동
→ [프로젝트에서 제거]: 이 프로젝트에서 제거
```

---

### 실습 포인트

- [ ] 메모리를 열어서 내용을 한글로 번역해본다
- [ ] 잘못되었거나 추가하고 싶은 내용이 있으면 직접 수정 요청해본다
- [ ] "이 프로젝트 주제에서 어긋난 대화를 찾아봐"를 입력해본다

## 실습 7 — 총무팀 전용 반복 업무 자동화

> **배우는 것:** 프로젝트를 활용하면 반복 업무를 체계적으로 관리할 수 있습니다.  
> 최동현 팀장만의 전담 총무 챗봇을 완성해봅니다.

### 총무팀 프로젝트 활용 시나리오

최동현 팀장이 앞으로 이 프로젝트를 어떻게 쓸 수 있는지 전체 그림입니다.

| 업무 | 활용 방법 | 예상 절약 시간 |
|------|----------|--------------|
| 월간 예산 보고서 | 집행 내역만 입력 → 표준 형식 자동 생성 | 2시간 → 30분 |
| 계약 검토 의견서 | 계약 개요 입력 → 검토 의견 초안 자동 생성 | 1시간 → 15분 |
| 비품 구매 요청 검토 | 요청 내용 입력 → 승인/반려 의견 자동 생성 | 30분 → 10분 |
| 공지문 작성 | 내용 키워드 입력 → 총무팀 스타일 공지문 생성 | 1시간 → 20분 |
| 회의록 정리 | 메모 붙여넣기 → 표준 형식 회의록 생성 | 1시간 → 15분 |

## 최동현의 프로젝트 활용 원칙 (실습 후 정리)

> 20년 경력 총무팀장의 AI 도구 사용 기준

1. **지침은 실제 업무 기준으로** — 평소 직원에게 설명하는 내용을 그대로 지침에 쓴다
2. **파일은 실제 양식을 올린다** — 빈 템플릿이라도 올려두면 형식이 정확해진다
3. **법적·재무적 판단은 검토 필수** — 클로드 결과물은 초안이다. 최종 서명 전 반드시 확인한다
4. **프로젝트는 주기적으로 정리한다** — 불필요한 채팅이 쌓이면 메모리도 흔들린다
5. **절차는 지침에 명시한다** — "500만원 이상 법무팀 검토" 같은 규칙은 반드시 지침에 넣는다
