## 실습 1 — 모호한 지시 vs 명확한 지시

> **배우는 것:** 프롬프트는 답변 품질을 결정짓는 가장 중요한 요소입니다.  
> 목적·대상·형식을 모두 담으면 훨씬 좋은 결과를 얻을 수 있습니다.

### 스토리

이수아는 신입사원 환영 이메일을 써달라고 클로드에 처음 입력해봤습니다.  
그런데 결과가 엉뚱하게 나왔어요. 왜 그랬을까요?

---

### ❌ 나쁜 프롬프트 (직접 입력해보세요)

```
이메일 써줘
```

**예상 결과:** 클로드가 "어떤 이메일인가요?" 라고 되묻거나, 엉뚱한 내용을 씁니다.

**이수아의 반응:** "AI가 이상하네... 내가 원하는 게 그게 아닌데."

---

### ✅ 좋은 프롬프트 (직접 입력해보세요)

```
KT 신입사원에게 보내는 환영 이메일을 써줘.
입사 첫날 일정을 포함해줘:
- 오전 9시: 인사팀 집합
- 오후: OJT 진행

말투는 따뜻하고 격식 있게, 200자 이내로 써줘.
```

**예상 결과:** 원하는 내용, 형식, 말투가 딱 맞게 나옵니다.

---

### 실습 포인트

- [ ] 나쁜 프롬프트를 클로드에 입력해보고 결과를 확인한다
- [ ] 좋은 프롬프트를 입력해보고 차이를 비교한다
- [ ] 좋은 프롬프트에서 **목적 / 대상 / 형식** 세 가지가 어디에 있는지 찾아본다

### 내가 받은 결과 메모

```
(여기에 클로드의 답변을 붙여넣으세요)
```

## 실습 2 — 역할 부여하기

> **배우는 것:** 역할(Role)을 부여하면 답변의 어조와 스타일을 원하는 대로 맞출 수 있습니다.

### 스토리

이수아는 같은 AI 교육 안내문을 두 버전으로 만들어야 합니다.  
- **신입사원용:** 긴장하지 않게, 친근하게  
- **팀장급용:** 간결하고 격식 있게  

내용은 똑같은데 말투만 달라야 해요. 어떻게 할까요?

---

### 버전 A — 신입사원용 (직접 입력해보세요)

```
너는 친근하고 다정한 선배 HR 담당자야.
신입사원이 긴장하지 않도록 KT AI 활용 교육 일정을 안내해줘.
말투는 편안하고 따뜻하게 써줘.

교육 일정:
- 날짜: 다음 주 화요일 오후 2시
- 장소: 본사 3층 대회의실
- 내용: 클로드 기초 실습
```

---

### 버전 B — 팀장급용 (직접 입력해보세요)

```
너는 격식 있는 HR 전문가야.
팀장급 이상을 대상으로 AI 교육 일정을 공지해줘.
간결하고 전문적인 말투로 써줘.

교육 일정:
- 날짜: 다음 주 화요일 오후 2시
- 장소: 본사 3층 대회의실
- 내용: 클로드 기초 실습
```

---

### 실습 포인트

- [ ] 두 버전을 모두 입력해서 결과를 비교한다
- [ ] 역할 부분만 바꾸면 결과가 어떻게 달라지는지 확인한다
- [ ] 역할 없이 같은 내용을 입력하면 어떻게 나오는지도 비교해본다

### 두 버전 비교 메모

| 항목 | 신입사원용 | 팀장급용 |
|------|-----------|---------|
| 첫 문장 | | |
| 말투 느낌 | | |
| 길이 | | |

## 실습 3 — 출력 형태 지정하기

> **배우는 것:** 형식을 지정하지 않으면 클로드가 임의로 판단합니다.  
> 표·번호목록·글머리기호 등 원하는 형태를 명시하면 바로 활용 가능한 결과물이 나옵니다.

### 스토리

이수아는 신입사원 온보딩 체크리스트를 만들어야 합니다.  
- 팀장 **최동현 부장**에게 보고할 때는 **표** 형식이 필요하고  
- 신입사원에게 나눠줄 때는 **번호 목록** 형식이 필요합니다.

같은 내용, 다른 형식!

---

### 버전 A — 보고용 표 (직접 입력해보세요)

```
신입사원 온보딩 첫 주 체크리스트를 표로 만들어줘.
열 구성: 항목 / 담당 / 완료 기한

담당자 정보:
- 사원증·출입카드: 총무팀 최동현
- 시스템 계정: IT팀
- 보안 서약서: 인사팀 이수아
- 부서 배치 안내: 인사팀 이수아
- OJT 일정 확인: 각 부서 담당자
```

---

### 버전 B — 직원 배포용 번호 목록 (직접 입력해보세요)

```
같은 내용을 신입사원이 직접 체크할 수 있는 번호 목록으로 만들어줘.
5개 항목으로, 신입사원 입장에서 행동하는 형식으로 써줘.
```

---

### 보너스 실습 — 형식 없이 넣으면?

```
신입사원 온보딩 첫 주 체크리스트 만들어줘.
```

> 어떻게 나오나요? 형식을 지정했을 때와 비교해보세요.

---

### 실습 포인트

- [ ] 표 버전과 번호 목록 버전을 모두 만들어본다
- [ ] 형식 미지정 버전도 입력해서 차이를 확인한다
- [ ] 실제 업무에서 어떤 형식이 언제 필요할지 생각해본다

## 실습 4 — 예시 제공하기

> **배우는 것:** 말로 설명하기 어려운 스타일은 예시를 보여주는 것이 훨씬 효과적입니다.  
> 클로드는 예시의 패턴을 파악하기 때문에 2~5개 예시가 적당합니다.

### 스토리

이수아는 매달 사내 공지를 씁니다.  
KT 인사팀 특유의 문체가 있는데, 클로드가 그 스타일 그대로 써주길 원합니다.  
"우리 팀 말투"를 어떻게 알려줄 수 있을까요?

---

### 실습 프롬프트 (직접 입력해보세요)

```
아래 예시 공지 스타일을 참고해서 '7월 AI 교육 안내' 공지문을 써줘.

[예시 1]
안녕하세요, 인사팀입니다.
6월 정기 교육 일정을 안내드립니다.
많은 참여 부탁드립니다.
감사합니다.

[예시 2]
안녕하세요, 인사팀입니다.
5월 복지포인트 신청 기간을 안내드립니다.
기간 내 신청 부탁드리며, 문의는 인사팀으로 연락 주세요.
감사합니다.

위 형식과 말투를 유지해서 7월 AI 교육 안내 공지문을 작성해줘.
교육 날짜는 7월 15일 화요일 오후 2시, 장소는 본사 3층 대회의실이야.
```

---

### 비교 실습 — 예시 없이 넣으면?

```
7월 AI 교육 안내 공지문을 써줘.
교육 날짜는 7월 15일 화요일 오후 2시, 장소는 본사 3층 대회의실이야.
```

> 예시가 있을 때와 없을 때, 스타일이 얼마나 달라지나요?

---

### 실습 포인트

- [ ] 예시 있는 버전과 없는 버전을 모두 입력해서 비교한다
- [ ] 실제 우리 팀에서 쓰는 공지 예시를 찾아 프롬프트에 추가해본다
- [ ] 예시를 1개만 넣었을 때와 2개 넣었을 때 차이가 있는지 확인한다

## 실습 5 — 나만의 HR 비서 세팅하기

> **배우는 것:** 클로드 설정에서 개인 맞춤 지침을 입력하면, 매번 역할을 지정하지 않아도  
> 내 스타일에 딱 맞는 답변이 자동으로 나옵니다.

### 스토리

매번 "너는 인사팀 담당자야, 존댓말로, 짧게..." 를 반복하던 이수아.  
설정 화면을 발견하고 한 번만 입력해두기로 합니다.

---

### 설정 방법

1. 클로드 화면 왼쪽 아래 프로필 클릭
2. **[설정]** 메뉴 클릭 (단축키: `Ctrl + ,`)
3. 왼쪽 **[일반]** 메뉴 선택
4. 호칭, 직무, 개인 맞춤 설정 입력 후 **[변경사항 저장]** 클릭

---

### 이수아가 입력할 맞춤 설정 예시

```
- 나는 KT 인사팀 이수아입니다.
- 답변은 항상 존댓말로 해주세요.
- 공지문·이메일을 자주 씁니다. 형식은 짧고 명확하게 해주세요.
- 노동법·취업규칙 관련 내용은 반드시 마지막에 "※ 세부 사항은 인사팀 확인이 필요합니다." 문구를 붙여주세요.
- 모든 공지문 마지막에는 "감사합니다." 로 마무리해주세요.
```

---

### 설정 전 vs 설정 후 비교

**설정 전 — 매번 길게 써야 함:**
```
너는 KT 인사팀 담당자야. 존댓말로, 짧게, 마지막에 감사합니다 붙여서
육아휴직 신청 안내 공지 써줘.
```

**설정 후 — 짧게만 써도 OK:**
```
육아휴직 신청 안내 공지 써줘.
```

> 설정을 마치면 클로드가 이수아의 스타일을 이미 알고 있어서 짧은 지시만으로도 맞춤 답변이 나옵니다.

---

### 실습 포인트

- [ ] 설정 화면을 열어 호칭과 직무를 입력해본다
- [ ] 내가 자주 쓰는 지침 3가지를 개인 맞춤 설정에 입력한다
- [ ] 설정 전후로 같은 프롬프트를 입력해서 차이를 확인한다

## 핵심 요약 (교재 01.4)

> ✅ 프롬프트 엔지니어링은 LLM의 답변 품질을 올리기 위해 입력을 최적화하는 기술입니다.  
> ✅ 컨텍스트 엔지니어링은 프롬프트와 첨부 파일, 대화 이력 등을 포함한 전체 정보를 최적화하는 기술입니다.  
> ✅ 프롬프트를 작성할 때 역할 부여, 출력 형태 지정, 예시 제공, 답변 분량 제한을 활용하면 도움이 됩니다.  
> ✅ 모델이 똑똑해질수록 프롬프트는 단순해지고, 컨텍스트는 복잡해집니다.
