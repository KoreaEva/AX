## 오늘의 배경

> 최동현 팀장의 고민들:
>
> - 매번 엑셀로 계약 만료일을 관리하는데 **한눈에 D-day가 안 보인다**
> - 회의실 사용 신청서를 이메일로 받으면 **취합이 너무 번거롭다**
> - 신입사원들이 온보딩 체크리스트를 **종이로 받아서 분실**하는 경우가 많다
> - 팀원들과 함께 쓸 수 있는 **업무 집중 타이머**가 필요하다
>
> 이수아가 말했습니다: *"팀장님, 그거 다 클로드로 만들 수 있어요. 코딩 몰라도요."*
>
> 최동현의 반응: *"...진짜로?"*

---

## 배포란 무엇인가 — 최동현식 이해

> 앱을 만들었다면, 다음 단계는 다른 사람들이 쓸 수 있도록 인터넷에 올리는 겁니다.  
> 이것을 **배포**라고 합니다.  
> 마치 집을 짓고 나서 주소를 부여하는 것과 같습니다.  
> 주소가 있어야 사람들이 찾아올 수 있듯이, 배포를 해야 동료들이 만든 앱을 쓸 수 있습니다.

| 구분 | 예전 방식 | 클로드 아티팩트 |
|------|---------|--------------|
| 서버 준비 | 필요 (복잡) | 불필요 |
| 도메인 구매 | 필요 (유료) | 불필요 |
| 코딩 지식 | 필수 | 불필요 |
| 배포 방법 | 개발자에게 의뢰 | [게시] 버튼 클릭 한 번 |
| 공유 방법 | 복잡한 설치 과정 | 링크 복사 후 카카오톡/이메일 전송 |
| 업데이트 | 개발자 재의뢰 | 클로드에게 수정 요청 후 재게시 |

> **주의:** 배포된 앱은 링크를 아는 누구나 접근할 수 있습니다.  
> 개인정보나 민감한 내용은 포함하지 않도록 주의하세요.

---

## 아티팩트 시작하는 방법

```
방법 1 — 아티팩트 메뉴에서:
왼쪽 사이드바 [Artifacts] 클릭 → [새 아티팩트] → [처음부터 시작하기]

방법 2 — 일반 채팅에서:
프롬프트에 "아티팩트로 만들어줘"라고 명시하면 자동으로 아티팩트 패널에 생성

카테고리 선택:
- 앱 및 웹사이트: 실제 작동하는 웹앱
- 생산성 도구: 타이머, 체크리스트, 계산기
- 퀴즈 또는 설문조사: 테스트, 설문
- 처음부터 시작하기: 직접 설명
```

> **TIP:** 소스 코드가 채팅창에 텍스트로 출력되면 아티팩트가 아닙니다.  
> 즉시 생성을 중단하고 "아티팩트로 다시 만들어줘"라고 지시하세요.

## 실습 1 — 계약 만료 D-day 카운터 만들기

> **배우는 것:** 가장 간단한 앱부터 만들어봅니다.  
> 엑셀에서 매번 날짜 계산하던 것을 웹앱으로 한 번에 해결합니다.

### 스토리

최동현 팀장은 매달 계약 만료일이 다가오는 계약들을 놓칠까봐 불안합니다.  
엑셀에서 날짜 함수로 계산하면 되지만, 한눈에 D-day가 보이는 전용 앱이 있으면 훨씬 편할 것 같습니다.

---

### 직접 입력해보세요

> [Artifacts → 새 아티팩트 → 생산성 도구] 선택 후 아래 입력

```
계약 만료 D-day 카운터 앱을 만들어줘.

기능:
- 계약명과 만료일을 입력하면 D-day가 자동 계산
- 여러 개의 계약을 동시에 관리 (추가/삭제 가능)
- D-day가 30일 이내이면 빨간색, 60일 이내이면 주황색, 그 외는 초록색으로 표시
- 가장 급한 계약이 위에 오도록 정렬

언어는 한국어로 해줘.
```

**앱 완성 후 테스트:**

| 계약명 | 만료일 | 확인 사항 |
|--------|--------|---------|
| (주)클린서비스 청소 용역 | 내년 7월 31일 | D-day 숫자 맞나요? |
| 복합기 임대 계약 | 이번 달 말일 | 빨간색으로 표시되나요? |
| 사옥 임대차 계약 | 3개월 뒤 | 주황색으로 표시되나요? |

- [ ] 계약 추가가 잘 됩니까?
- [ ] D-day 숫자가 정확하게 계산됩니까?
- [ ] 색상 구분이 제대로 적용됩니까?
- [ ] 만료일 순으로 정렬이 됩니까?

---

### 기능 추가 요청 (직접 입력해보세요)

기본 기능이 작동하면 원하는 기능을 추가합니다.

```
기능을 추가해줘:
- D-day 30일 이내 계약은 상단에 "주의 필요" 배너로 강조 표시
- 전체 계약 수와 만료 임박 계약 수를 상단 요약으로 표시
- 계약 담당자 이름도 입력할 수 있게 해줘
```

---

### 내가 받은 결과 메모

```
앱에서 잘 된 점:

추가로 수정하고 싶은 점:

실제 업무에 바로 쓸 수 있나요? (Y/N):
```

## 실습 2 — 업무 집중 포모도로 타이머 만들기

> **배우는 것:** 포모도로 기법(25분 집중 + 5분 휴식)으로 업무 효율을 높이는 타이머를 만듭니다.  
> 만든 후 디자인까지 수정해서 "AI가 만든 티"를 없애봅니다.

### 스토리

최동현 팀장은 총무팀 회의에서 팀원들의 집중력 문제를 논의했습니다.  
이수아가 포모도로 기법을 추천했고, 팀 전체가 함께 쓸 수 있는 타이머를 만들기로 했습니다.

---

### 직접 입력해보세요 — 타이머 생성

> [Artifacts → 새 아티팩트 → 생산성 도구] 선택 후 아래 입력

```
포모도로 타이머 만들어줘.
언어는 한국어로 해줘.
```

**타이머 작동 확인:**
- [ ] [시작] 버튼을 누르면 25:00부터 카운트다운이 시작됩니까?
- [ ] 25분이 지나면 휴식 모드로 자동 전환됩니까?
- [ ] 완료된 포모도로 횟수가 표시됩니까?

---

### 디자인 수정 요청 (직접 입력해보세요)

기본 기능은 작동하지만 디자인이 마음에 안 들면 클로드에게 요청합니다.

```
디자인이 마음에 안 들어.
깔끔하고 심플한 미니멀 스타일로 바꿔줘.
색상은 KT 브랜드 컬러인 빨간색 계열로 포인트를 줘.
```

또는

```
노션처럼 깔끔하고 심플한 스타일로 바꿔줘.
```

또는

```
애플 제품처럼 모던하고 세련된 스타일로 바꿔줘.
```

**수정 전후 비교:**

| 항목 | 수정 전 | 수정 후 |
|------|--------|--------|
| 전체 느낌 | | |
| 색상 | | |
| 버튼 스타일 | | |
| AI가 만든 티 | 많이 남 / 조금 남 / 없음 | |

---

### 기능 커스터마이징 (직접 입력해보세요)

```
기능을 수정해줘:
- 집중 시간을 25분 외에 직접 입력해서 바꿀 수 있게 해줘
- 타이머 완료 시 소리 알림 추가
- 오늘 총 집중 시간을 분 단위로 표시해줘
```

## 실습 3 — 온보딩 체크리스트 앱 만들고 배포하기

> **배우는 것:** 신입사원이 스마트폰에서 바로 쓸 수 있는 체크리스트 앱을 만들고,  
> 링크로 배포해서 누구나 접근할 수 있게 합니다.

### 스토리

최동현 팀장은 신입사원들이 온보딩 체크리스트를 종이로 받으면 분실하는 경우가 많다는 걸 알았습니다.  
스마트폰에서 바로 쓸 수 있는 앱을 만들어 링크로 공유하면 훨씬 편리할 것 같습니다.

---

### 직접 입력해보세요 — 앱 생성

> [Artifacts → 새 아티팩트 → 처음부터 시작하기] 선택 후 아래 입력

```
신입사원 온보딩 체크리스트 앱을 만들어줘.

체크리스트 항목:
입사 첫날:
- 인사팀 오리엔테이션 참석
- 사원증 및 출입카드 수령 (총무팀)
- 사내 이메일 계정 설정 확인
- 보안 서약서 서명

입사 1주차:
- 부서 배치 및 팀장 면담 완료
- 시스템 접근 권한 신청 (IT팀)
- 사내 규정집 열람
- OJT 일정 확인

입사 1개월:
- 중간 점검 면담 (인사팀 이수아)
- 업무 파악 보고서 제출

기능:
- 항목 체크/해제 기능
- 전체 진행률 퍼센트로 표시
- 완료된 항목은 회색으로 처리
- 이름 입력 칸 (예: 홍길동의 온보딩 체크리스트)

언어는 한국어로 해줘.
```

**앱 테스트:**
- [ ] 항목을 클릭하면 체크가 됩니까?
- [ ] 체크할수록 진행률 숫자가 올라갑니까?
- [ ] 이름 입력 후 제목이 바뀝니까?

---

### 배포하기 — 링크 생성 및 공유

앱이 완성되면 인터넷에 배포해서 누구나 접근할 수 있게 만듭니다.

```
배포 방법:
1. 아티팩트 패널 오른쪽 상단 [...] 버튼 클릭
2. [게시] 선택
3. [게시 및 링크 복사] 클릭
4. 생성된 링크 복사
5. 카카오톡/이메일로 신입사원에게 전송
```

**배포 후 확인:**
- [ ] 게시 완료 후 링크가 생성되었나요?
- [ ] 브라우저에서 링크를 열어 앱이 정상 작동하나요?
- [ ] 스마트폰에서도 잘 표시되나요?

> **주의:** 한 번 게시했다가 비공개로 바꾼 아티팩트는 다시 게시할 수 없습니다.  
> 다시 게시하고 싶다면 클로드에게 작은 수정을 지시해 새 버전을 만든 뒤 게시하세요.

## 실습 4 — 업무 스타일 진단 테스트 만들기

> **배우는 것:** 질문과 선택지로 구성된 심리 테스트 앱을 만듭니다.  
> 팀 워크숍이나 신입사원 교육에서 아이스브레이킹 도구로 활용할 수 있습니다.

### 스토리

최동현 팀장은 다음 달 팀 워크숍에서 팀원들의 업무 스타일을 파악하는 시간을 갖기로 했습니다.  
종이 설문지 대신, 스마트폰에서 바로 할 수 있는 테스트 앱을 만들기로 했습니다.

---

### 단계별 제작 방법

**1단계 — 먼저 질문을 확인한다 (직접 입력해보세요)**

```
총무팀 팀원들의 업무 스타일을 파악하는 심리 테스트가 콘셉트야.
질문을 6개로 만들 거야. 아티팩트를 제작하기 전에, 질문 예시를 미리 만들어봐.

테스트 주제: 나는 어떤 업무 유형일까?
결과 유형 4가지:
- 계획형 (꼼꼼하게 준비하는 타입)
- 추진형 (빠르게 실행하는 타입)
- 협력형 (소통을 중시하는 타입)
- 분석형 (데이터와 근거를 중시하는 타입)
```

**2단계 — 질문이 마음에 들면 앱으로 만든다 (직접 입력해보세요)**

```
방금 질문으로 만들어줘. 언어는 한국어로 해줘.
```

**3단계 — 완성된 앱 확인**
- [ ] 질문이 한국어로 표시되나요?
- [ ] 선택지를 클릭하면 다음 질문으로 넘어가나요?
- [ ] 마지막에 결과 유형이 나오나요?
- [ ] 결과에 설명과 특징이 포함되어 있나요?

---

### 테스트 앱 배포 및 공유 (직접 입력해보세요)

```
배포 → 게시 → 게시 및 링크 복사
```

```
이 링크를 팀원들에게 카카오톡으로 공유:
"[팀 워크숍 사전 준비] 나는 어떤 업무 유형일까? 테스트 해보세요: [링크]"
```

## 실습 5 — 나만의 총무팀 전용 앱 직접 설계하기

> **배우는 것:** 지금까지 배운 것을 응용해 실제 업무에서 필요한 나만의 앱을 직접 기획하고 만듭니다.

### 총무팀에서 만들 수 있는 앱 아이디어

아래 중 하나를 골라 직접 프롬프트를 작성해보세요.

---

**아이디어 A: 회의실 예약 현황판**

```
(여기에 직접 프롬프트를 작성해보세요)

힌트:
- 회의실 이름, 시간대, 예약자 입력
- 시간대별 예약 현황을 한눈에 표시

```

---

**아이디어 B: 비품 재고 관리 앱**

## 실습 6 — 스마트폰에서 아티팩트 확인하기

> **배우는 것:** PC에서 만든 아티팩트를 스마트폰에서 그대로 사용합니다.  
> 클로드 계정이 연동되어 있으면 모든 기기에서 내 아티팩트를 볼 수 있습니다.

### 스마트폰 앱에서 확인하는 방법

```
1. 스마트폰에 클로드 앱 설치 (App Store / Google Play)

4. PC에서 만든 아티팩트 목록이 모두 보임
5. 원하는 앱 선택 → 바로 실행
```

**스마트폰에서 확인하기:**
- [ ] 스마트폰 클로드 앱에서 [아티팩트] 메뉴를 찾았나요?

## 아티팩트 제작 시 주의사항

**자주 발생하는 오류와 해결법:**

| 문제 상황 | 해결 방법 |
|----------|---------|
| 소스 코드가 채팅창에 그대로 출력됨 | 즉시 생성 중단 → "아티팩트로 다시 만들어줘" |
| 앱이 영어로 만들어짐 | 프롬프트에 "언어는 한국어로 해줘" 추가 |
| 기능이 작동하지 않음 | "이 기능이 작동하지 않아. 수정해줘" |
| 디자인이 마음에 안 듦 | "노션 스타일로", "미니멀하게", "KT 빨간색 계열로" 등 구체적으로 요청 |
| 오류가 반복됨 | 새 채팅에서 처음부터 다시 시도 |

**배포 관련 주의사항:**

- 배포된 앱은 링크를 아는 누구나 접근 가능합니다
- 개인정보(직원 이름, 연락처, 급여 등)는 절대 포함하지 마세요
- 민감한 계약 금액이나 내부 정보도 포함하지 마세요
- 한 번 게시 후 비공개 전환하면 다시 게시 불가 (새 버전 만들어야 함)

## 최동현의 앱 개발 5원칙

> 코딩 몰라도 앱을 만드는 총무팀장의 기준

1. **구체적으로 요청한다** — "앱 만들어줘"보다 기능을 하나씩 명시할수록 원하는 결과가 나온다
2. **먼저 작동 확인, 그다음 디자인** — 기능이 제대로 되는지 먼저 확인하고 디자인은 나중에 수정한다
3. **민감 정보는 넣지 않는다** — 배포된 앱은 누구나 볼 수 있다는 걸 항상 기억한다
4. **수정은 말로 하면 된다** — 마음에 안 드는 부분은 말로 설명하면 클로드가 고쳐준다
5. **만든 앱은 [아티팩트]에 저장된다** — 언제든 다시 꺼내 수정하거나 배포할 수 있다
