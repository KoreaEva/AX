> 숫자만 있으니까 한눈에 안 들어와요. 차트랑 흐름도 같이 넣어서 보기 좋게 만들어봐요.

## 비주얼라이저 vs 아티팩트 — 뭐가 다른가요?

| 구분 | 비주얼라이저 | 아티팩트 |
|------|------------|---------|
| 위치 | 대화 중간에 인라인으로 삽입 | 오른쪽 별도 패널에 표시 |
| 특징 | 대화 흐름 속에 자연스럽게 녹아듦 | 독립적인 작업 공간처럼 작동 |
| 저장 | 차트를 파일/클립보드로 저장 | PDF 다운로드, 아티팩트로 저장 가능 |
| 적합한 용도 | 데이터 차트, 인터랙티브 위젯 | 긴 문서, 웹페이지, 다이어그램 |
| 수정 방법 | 클로드에게 수정 요청 | 클로드에게 수정 요청 후 패널 업데이트 |

> 이수아의 정리: "차트 하나 빠르게 보려면 비주얼라이저, 보고서 문서 전체를 만들려면 아티팩트!"

## 실습 1 — 도넛 차트로 채용 예산 구조 한눈에 보기

> **배우는 것:** 비주얼라이저로 비율 데이터를 도넛 차트로 즉시 시각화합니다.  
> 프롬프트 한 줄이면 클로드가 알아서 비율을 계산해 깔끔한 차트로 만들어 줍니다.

### 스토리

이수아는 올해 채용 예산이 어디에 얼마나 쓰였는지 최팀장에게 보고해야 합니다.  
숫자 표만 있으면 한눈에 안 들어와서 도넛 차트가 필요합니다.

---

### 실습 1-A: 가상 데이터로 먼저 연습 (직접 입력해보세요)

```
올해 KT 인사팀 채용 예산 3,000만원의 항목별 지출 구조를
도넛 차트로 시각화해줘. 상세 항목은 가상의 데이터를 사용해.
```

**차트가 생성된 후 확인 포인트:**
- [ ] 도넛 차트가 대화 화면 안에 바로 표시되었나요?
- [ ] 각 항목에 마우스를 올리면 금액과 비율이 툴팁으로 나타나나요?
- [ ] 차트 위에 항목별 색상과 비율이 범례로 정리되어 있나요?

---

### 실습 1-B: 실제 데이터로 다시 만들기 (직접 입력해보세요)

```
KT 인사팀 2025년 채용 예산 집행 현황을 도넛 차트로 시각화해줘.

항목별 금액:
- 채용 플랫폼 광고비: 800만원
- 헤드헌터 수수료: 600만원
- 인적성 검사 비용: 400만원
- 면접 운영비 (장소·식비): 350만원
- 입사자 온보딩 교육비: 500만원
- 기타 행정 비용: 350만원
총 예산: 3,000만원
```

**차트 저장 방법:**
```
차트에 마우스를 올리면 [...] 버튼 클릭
→ Copy to clipboard: 클립보드에 복사
→ Download file: 파일로 저장
→ Save as artifact: 아티팩트로 저장
```

---

### 내가 받은 결과 메모

```
가상 데이터 차트에서 가장 큰 비중을 차지한 항목:

실제 데이터 차트에서 예상과 달랐던 점:

최팀장 보고에 활용할 수 있을 것 같나요? (Y/N):
```

## 실습 2 — 꺾은선 차트로 월별 채용 현황 추이 보기

> **배우는 것:** 시간의 흐름에 따라 달라지는 데이터는 꺾은선 차트가 적절합니다.  
> 월별 지원자 수, 합격률, 입사율 추이를 한눈에 파악할 수 있습니다.

### 스토리

이수아는 올해 월별 채용 현황을 추이로 보여달라는 요청을 받았습니다.  
매달 지원자 수, 서류 합격자 수, 최종 입사자 수가 어떻게 변했는지 차트로 만들어야 합니다.

---

### 직접 입력해보세요

```
아래 데이터로 월별 채용 현황 추이를 꺾은선 차트로 만들어줘.
세 개의 선(지원자 수, 서류 합격자 수, 최종 입사자 수)을 한 차트에 표시해줘.

월별 데이터:
1월: 지원자 45명, 서류합격 12명, 입사 3명
2월: 지원자 38명, 서류합격 10명, 입사 2명
3월: 지원자 72명, 서류합격 20명, 입사 6명
4월: 지원자 89명, 서류합격 25명, 입사 8명
5월: 지원자 61명, 서류합격 18명, 입사 5명
6월: 지원자 95명, 서류합격 28명, 입사 9명
```

**차트 확인 포인트:**
- [ ] 세 개의 선이 구분되는 색상으로 표시되었나요?
- [ ] 마우스를 올리면 해당 월의 상세 수치가 나타나나요?
- [ ] 지원자가 가장 많은 달과 입사자가 가장 많은 달이 같은가요?

---

### 추가 질문으로 분석 심화하기 (직접 입력해보세요)

차트를 보고 난 뒤 분석을 요청합니다.

```
위 채용 현황 차트를 분석해줘.
- 지원자 대비 최종 입사 전환율이 가장 높은 달은 언제인지
- 채용이 가장 활발한 분기는 언제인지
- 개선이 필요한 구간은 어디인지
간략하게 정리해줘.
```

## 실습 3 — 인터랙티브 레이더 차트로 신입사원 역량 평가하기

> **배우는 것:** 슬라이더로 값을 직접 조정하면서 차트에 실시간 반영되는 인터랙티브 위젯을 만듭니다.  
> 면접관이 항목별 점수를 조정하면 차트가 즉시 바뀌는 평가 도구를 만들 수 있습니다.

### 스토리

이수아는 신입사원 면접 평가 결과를 시각화하고 싶습니다.  
역량별 점수를 슬라이더로 조정하면 레이더 차트가 실시간으로 반영되면,  
면접관들이 회의 중에 바로 비교해볼 수 있을 것 같습니다.

---

### 직접 입력해보세요

```
신입사원 면접 역량 평가를 레이더 차트로 만들어줘.
슬라이더로 각 역량 점수를 조정하면 차트에도 실시간 반영되게 만들어줘.
이상적인 신입사원 역량 기준선도 함께 겹쳐서 그려줘.

평가 항목:
- 직무 전문성 (0~100)
- 커뮤니케이션 능력 (0~100)
- 팀워크 및 협업 (0~100)
- 문제 해결력 (0~100)
- 적극성 및 주도성 (0~100)
- 조직 적응력 (0~100)
```

**인터랙티브 차트 사용해보기:**
- [ ] 슬라이더를 움직이면 차트가 실시간으로 변하나요?
- [ ] 이상적인 기준선과 내 평가 점수의 차이가 한눈에 보이나요?
- [ ] 가장 점수 차이가 큰 항목이 어디인지 바로 파악되나요?

---

### 활용 아이디어

이 인터랙티브 차트는 다양한 HR 상황에 응용할 수 있습니다.

```
응용 프롬프트 1 — 지원자 두 명 비교:
지원자 A와 지원자 B의 면접 역량을 비교하는 레이더 차트를 만들어줘.
두 사람의 점수를 다른 색으로 표시하고, 슬라이더로 각 점수를 조정하면
차트가 실시간 반영되게 해줘.
```

```
응용 프롬프트 2 — 팀별 교육 이수율 비교:
부서별 AI 역량 교육 이수율을 레이더 차트로 시각화해줘.
부서: 인사팀, 총무팀, 네트워크운용팀, 법무팀, 마케팅팀
각 부서 이수율은 가상의 데이터를 사용해줘.
```

## 실습 4 — 아티팩트로 온보딩 절차 흐름도 만들기

> **배우는 것:** 아티팩트 패널에서 머메이드 다이어그램으로 업무 프로세스를 시각화합니다.  
> 복잡한 절차를 말로 설명하는 것보다 흐름도 하나가 훨씬 명확합니다.

### 스토리

이수아는 신입사원 온보딩 절차를 매번 말로 설명하느라 힘들었습니다.  
한 번 흐름도로 만들어두면 앞으로 신입사원이 입사할 때마다 보여주면 됩니다.

---

### 직접 입력해보세요

```
신입사원 온보딩 절차를 Mermaid 코드로 .mermaid 파일 아티팩트로 만들어줘.

온보딩 절차:
1. 인사팀 집합 및 오리엔테이션
2. 사원증 및 출입카드 발급 (총무팀)
3. 시스템 계정 생성 및 이메일 설정 (IT팀)
4. 보안 서약서 서명 (인사팀)
5. 부서 배치 및 팀장 면담
6. OJT 시작 (1주차: 업무 파악)
7. 중간 점검 면담 (3개월차, 인사팀)
8. 정식 배치 확정 (6개월차)

각 단계에서 담당 부서를 함께 표시해줘.
```

**아티팩트 패널 확인:**
- [ ] 화면 오른쪽에 아티팩트 패널이 열렸나요?
- [ ] 온보딩 절차가 흐름도로 표시되었나요?
- [ ] 각 단계별 담당 부서가 표시되어 있나요?

> **TIP:** 클로드가 아티팩트 패널이 아닌 대화 공간에 바로 그렸다면  
> "머메이드로 아티팩트 패널에 다시 그려줘"라고 지시하세요.

---

### 다이어그램 수정하기 (직접 입력해보세요)

아티팩트를 수정하려면 클로드에게 말로 요청합니다.

```
온보딩 흐름도에서 3개월 중간 점검 단계 다음에
"적응 지원 프로그램 안내 (선택사항)"을 추가해줘.
그리고 시스템 계정 생성 단계와 보안 서약서 서명 단계를 병렬로 처리할 수 있게 바꿔줘.
```

- [ ] 수정 요청한 부분만 바뀌고 나머지는 그대로 유지되었나요?

---

### draw.io로 옮겨서 더 다듬기

클로드가 만든 머메이드 코드를 draw.io에서 더 예쁘게 편집할 수 있습니다.

```
방법:
1. 아티팩트 패널 위쪽 [복사] 버튼 클릭
2. 브라우저에서 draw.io 접속
3. [배치 → 삽입 → Mermaid...] 클릭
4. 복사한 코드 붙여넣기 → [삽입] 클릭
5. draw.io에서 색상, 폰트, 레이아웃 자유롭게 편집
```

## 실습 5 — 아티팩트로 채용 공고 PDF 보고서 만들기

> **배우는 것:** 아티팩트로 생성한 문서를 PDF로 다운로드해 공유하거나 보관합니다.  
> 포맷이 깨질 걱정 없이 어떤 기기에서든 같은 모양으로 보입니다.

### 스토리

이수아는 상반기 채용 결과 보고서를 PDF로 만들어 최팀장에게 제출해야 합니다.  
아티팩트로 만들면 Word를 열지 않아도 깔끔한 문서를 바로 PDF로 저장할 수 있습니다.

---

### 직접 입력해보세요 — 보고서 초안 생성

```
상반기 채용 결과 보고서를 아티팩트에 작성해줘.
전문적인 비즈니스 문서 형식으로, 각 섹션마다 제목을 붙여줘.

포함 내용:
- 채용 개요 (기간: 2025년 1월~6월, 목표 채용 인원: 20명)
- 채용 채널별 지원 현황 (가상 데이터 사용)
- 최종 채용 결과 (직군별 입사자 수)
- 채용 과정 주요 이슈 및 개선 사항
- 하반기 채용 계획 방향

작성일: 2025년 7월 1일
작성: KT 인사팀 이수아
```

**아티팩트 패널에서 확인:**
- [ ] 오른쪽 패널에 문서가 표시되었나요?
- [ ] 각 섹션마다 제목이 붙어 있나요?
- [ ] 마음에 들지 않는 부분이 있으면 수정을 요청해보세요

---

### 문서 수정 요청하기 (직접 입력해보세요)

```
보고서에서 채용 채널별 현황을 표 형식으로 바꿔줘.
열 구성은 채용 채널 / 지원자 수 / 서류 합격자 수 / 최종 입사자 수 / 전환율(%)로 해줘.
```

---

### PDF로 저장하기

```
이 결과를 PDF로 다운로드하고 싶어.
```

**PDF 저장 후 확인:**
- [ ] PDF 파일이 정상적으로 생성되었나요?
- [ ] 파일 이름과 형식을 확인하고 저장 위치를 지정했나요?
- [ ] PDF를 열었을 때 보고서 형식이 깨지지 않고 잘 보이나요?

> **주의:** PDF로 저장하기 전에 문서 내용을 꼼꼼히 확인하세요.  
> PDF 파일은 한 번 저장하면 직접 수정하기 어렵습니다.  
> 수정이 필요하면 클로드에게 다시 요청해 아티팩트를 업데이트한 후 새로 다운로드하세요.

## 실습 6 — 아티팩트로 교육 이수율 간트 차트 만들기

> **배우는 것:** 간트 차트로 교육 일정과 이수 현황을 시간순으로 시각화합니다.  
> 파워포인트에서 도형을 정렬하고 화살표를 연결하느라 30분을 쓰던 시절은 이제 지났습니다.

### 스토리

이수아는 하반기 교육 일정을 최팀장에게 보고해야 합니다.  
어떤 교육이 몇 주차에 진행되는지 한눈에 보이는 간트 차트가 필요합니다.

---

```
KT 인사팀 하반기 사내 교육 일정을 간트 차트로 아티팩트에 만들어줘.

교육 일정:
- 1~2주차: AI 기초 활용 교육 (전 직원 필수)

- 4주차: 정보보안 의무 교육 (전 직원 필수)
- 5~6주차: 리더십 역량 강화 과정 (팀장급 이상)
- 7주차: 직무별 심화 교육 (선택)
- 8주차: 교육 결과 평가 및 수료식

담당: 인사팀 이수아
기간: 7월 ~ 8월
```

## 아티팩트에서 쓸 수 있는 파일 형식
