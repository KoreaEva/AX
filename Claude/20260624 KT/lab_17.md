## 실습 1 🧭 필요한 스킬 찾는 방법 & 품질 평가 기준

### 📌 상황 설명

준혁은 신규 네트워크 장비 도입 프로젝트를 맡게 됐다.  
기술 검토는 자신 있지만, 경영진에게 제출할 프로젝트 기획서와 비용 타당성 분석은 처음이다.  
직접 스킬을 만들기 전에, 전 세계 전문가들이 이미 만들어둔 스킬을 먼저 찾아보기로 했다.

### 🌐 스킬 검색 사이트

| 사이트 | 특징 |
|--------|------|
| **skillsmp.com** | 디자인이 친절하고 검색이 쉬움. 지금 실습에 사용 |
| **skills.sh** | 클로드 코드에서 사용하기 편리. 개발 작업에 유용 |

### ✅ 좋은 스킬을 고르는 평가 기준

스킬을 다운로드하기 전에 반드시 아래 항목을 확인하세요.  
얼굴도 모르는 누군가가 만든 소스 코드를 내 컴퓨터에서 실행하는 일이기 때문입니다.

**1️⃣ 문서화 수준 확인**

좋은 스킬은 아래 내용을 빠짐없이 제공합니다.

- 스킬이 무엇을 하는지 명확한 설명
- 사용 방법과 예시
- 필요한 입력 파일 형식
- 기대할 수 있는 출력 결과
- 알려진 제약 조건이나 주의사항

**2️⃣ 사용자 피드백 확인**

- ⭐ Stars 개수가 많을수록 신뢰도 높음
- 업데이트가 최근에 이루어진 저장소 선택
- 실제로 잘 작동하는지, 문제가 보고되어 있지 않은지 확인

**3️⃣ 보안 확인**

- 스킬이 어떤 데이터에 접근하는지 확인
- 파일 시스템 접근, 외부 서버 통신, 민감한 정보를 처리하는 스킬은 **신뢰할 수 있는 출처**에서만 설치

### 💡 준혁의 한마디
> "개발자 눈으로 보니까 Stars 숫자랑 마지막 업데이트 날짜가 제일 먼저 눈에 들어오더라고요. 코드 검토하는 것처럼 평가하면 돼서 오히려 편했어요."

## 실습 2 📋 project-planner 스킬로 네트워크 장비 도입 계획서 만들기

### 📌 상황 설명

준혁은 노후화된 KT 광화문 사옥 코어 스위치 교체 프로젝트를 기획해야 한다.  
기술 스펙 정리는 금방 하겠는데, 마일스톤·태스크 분해·리스크 관리까지 담긴 계획서는 부담스럽다.  
`project-planner` 스킬이 복잡한 프로젝트를 실행 가능한 작은 단위로 분해해준다는 걸 알게 됐다.

### 🔧 스킬 다운로드 & 설치 방법

**01단계** skillsmp.com에 접속합니다.  
검색창에 `Project-planner`를 입력해 검색합니다.  
**Shubhamsaboo**가 개발한 스킬을 찾습니다.

> ℹ️ Shubham Saboo는 구글 클라우드의 시니어 AI 프로덕트 매니저입니다. 신뢰할 만한 출처입니다.  
> Stars 97,000+로 약 10만 명의 개발자가 '좋아요'를 누른 검증된 스킬입니다.

**02단계** 찾은 스킬을 클릭하고 **[wget skill.zip]**을 눌러 압축 파일을 다운로드합니다.  
이 파일은 따로 압축을 해제할 필요가 없습니다.

**03단계** 클로드 데스크톱 앱을 열고 왼쪽 패널에서  
**[사용자 지정(Customize) → 스킬]**로 들어간 다음  
**[+ → 스킬 만들기 → 스킬 업로드]**를 클릭합니다.

**04단계** 스킬 업로드 창에 다운로드한 `.zip` 파일을 **드래그 앤 드롭**합니다.  
설치가 완료되면 스킬 목록에 `project-planner`가 표시됩니다.

> ℹ️ 스킬 파일은 `.zip`이나 `.skill`로 압축되어 있어야 합니다.  
> 인터넷에서 다운로드한 스킬은 `.zip`, 클로드에서 생성한 스킬은 `.skill`인 경우가 많습니다.

### 💬 프롬프트 예시

```
Project-planner와 DOCX 스킬을 사용해서
광화문 사옥 코어 스위치 교체 프로젝트 계획서를 만들어줘.
기간은 8주, 팀 구성은 네트워크 엔지니어 3명이야.
```

### ✅ 예상 결과 — 포함되는 내용

```
📄 KT 광화문 코어 스위치 교체 프로젝트 플랜

프로젝트 개요
- 목표: 노후 코어 스위치 교체 및 네트워크 안정성 강화
- 기간: 8주 (주당 15~20시간, 총 120~160시간)
- 팀원: 3인 (네트워크 엔지니어)
- 제약조건: 서비스 중단 최소화 / 야간 작업 필수

추천 진행 방식
- 현황 분석 → 장비 선정 → 도입 계획 → 테스트 → 절체 → 안정화

마일스톤 (4개)
1주차   현황 조사 및 장비 스펙 확정
3주차   장비 발주 및 사전 구성 완료
6주차   테스트 환경 검증 완료
8주차   실서비스 절체 및 안정화

Phase 1~4 상세 태스크
- 각 태스크별 공수, 선행 작업, 완료 기준 포함

리스크 & 대응 전략
- 절체 중 서비스 장애 발생 시 롤백 계획
- 조달 지연 시 임시 장비 운용 방안
```

### 💡 준혁의 한마디
> "기술 스펙 정리는 금방 하는데, 마일스톤이랑 리스크 항목이 항상 빠졌었어요. 스킬 쓰고 나서 팀장님한테 '이번엔 계획서가 제대로 나왔네'라는 말 처음 들었습니다."

## 실습 3 💰 creating-financial-models 스킬로 장비 도입 수익성 분석하기

### 📌 상황 설명

준혁은 코어 스위치 교체 비용이 3억 원인데, 경영진이 "ROI가 있느냐"고 물어왔다.  
재무 분석은 완전히 낯선 영역이다. 그런데 앤트로픽에서 만든 재무 모델링 스킬을 쓰면  
경제 지식이 없어도 전문가 수준의 수익성 분석이 가능하다는 걸 알게 됐다.

### 🔧 스킬 다운로드 & 설치 방법

**01단계** skillsmp.com에 접속해 `creating-financial-models`를 검색합니다.  
같은 이름의 스킬이 여러 개이므로 **제작자가 `anthropics`인지** 반드시 확인합니다.

> ⚠️ Stars 33,000+이며 앤트로픽 공식 저장소(`anthropics/claude-cookbooks`)에서 관리됩니다.

**02단계** 찾은 스킬을 클릭하고 **[wget skill.zip]**을 눌러 압축 파일을 다운로드합니다.

**03단계** 클로드 데스크톱 앱에서  
**[사용자 지정(Customize) → 스킬]**에 다운로드한 스킬을 업로드해 설치합니다.

### 💬 프롬프트 예시

```
creating-financial-models 스킬을 사용해서
KT 광화문 사옥 코어 스위치 교체 사업의 수익성을 분석해줘.
결과는 엑셀 워크북 형태로 만들어줘.

- 초기 투자비용: 3억 원 (장비 2.5억 + 공사 0.5억)
- 기대 효과: 장애 발생 건수 70% 감소, 유지보수 비용 연 4,000만 원 절감
- 분석 기간: 5년
```

### ✅ 예상 결과 — 엑셀 워크북 구성 (4개 시트)

**시트 1: 투자 개요**

| 항목 | 금액(만원) | 단위 | 비고 |
|------|-----------|------|------|
| 장비 구매비 | 25,000 | 만원 | 코어 스위치 2대 |
| 설치·공사비 | 5,000 | 만원 | 야간 작업 포함 |
| 초기 투자 합계 | 30,000 | 만원 | 자동 합산 |
| 연간 유지보수 절감 | 4,000 | 만원 | 장애 감소 효과 |
| 장애 복구 비용 절감 | 1,800 | 만원 | 건당 60만원 × 70% 감소 |

**시트 2: 월별 손익**

| 항목 | 1년차 | 2년차 | 3년차 | 합계 |
|------|-------|-------|-------|------|
| 비용 절감 효과 | 5,800 | 6,090 | 6,395 | 18,285 |
| 유지비용 | 1,200 | 1,200 | 1,200 | 3,600 |
| 순이익 | 4,600 | 4,890 | 5,195 | 14,685 |

**시트 3: 3년 수익 전망**

```
총 투자금액     : 30,000만원
1년차 손익분기  : 투자비 대비 19.3% 회수
투자 회수 기간  : 약 4.7년
5년 누적 ROI    : 약 128%
연 내부수익률(IRR): 21.4%
```

**시트 4: 민감도 분석**

```

구분          고객수(장애감소율)  객단가(절감단가)  ROI

기본 시나리오    70% 감소           건당 60만원      128%  ✅ 수익
보수 시나리오    50% 감소           건당 50만원       82%  ⚠️ 재검토
```

### 💡 준혁의 한마디
> "ROI가 뭔지도 제대로 몰랐는데, 스킬이 DCF 분석부터 민감도 분석까지 다 만들어줬어요. 경영진 발표에서 '재무 분석 수준이 높다'는 피드백 받았습니다. 사실 저는 숫자만 넣었는데요."
