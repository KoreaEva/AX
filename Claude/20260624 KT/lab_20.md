## 실습 1 🕘 스케줄링이란? 개념 이해하기

### 📌 상황 설명

준혁은 매일 같은 시간에 같은 작업을 반복한다.  
장애 이력 확인, 미처리 티켓 점검, 팀 문서 업데이트 확인.  
이런 반복 작업을 자동화하려면 보통 크론탭(crontab)이나 태스크 스케줄러를 써야 한다.  
하지만 코워크의 스케줄링 기능을 쓰면 명령어 없이 말로만 설정할 수 있다.

### 📌 스케줄링이란?

**스케줄링**은 정해진 주기에 정해진 작업을 반복적으로 수행하는 것입니다.  
모든 자동화의 시작이자 끝이죠.

준혁이 매일 쓰는 스케줄링 예시를 생각해보면 쉽습니다.

| 스케줄링 예시 | 주기 | 작업 |
|---------------|------|------|
| 알람 | 매일 오전 7:30 | 소리 울리기 |
| 네트워크 모니터링 | 매 5분 | 장비 상태 체크 |
| 백업 작업 | 매일 새벽 2시 | 파일 백업 수행 |
| **코워크 스케줄링** | **원하는 주기** | **말로 지시한 모든 작업** |

원래 스케줄링을 하려면 어려운 명령어를 입력해서 스케줄링 시스템을 구축해야 합니다.  
하지만 코워크만 있으면 그럴 필요가 없습니다.  
단순한 사용법만 익히면 바로 스케줄링 시스템을 구축할 수 있습니다.

> ⚠️ 스케줄링된 작업이 실행되려면 **컴퓨터가 켜져 있어야** 합니다.  
> 컴퓨터가 꺼져 있거나 절전 모드에 있으면 예약된 작업이 실행되지 않습니다.  
> 노트북 사용자는 이 점을 꼭 고려하세요.

### 💡 준혁의 한마디
> "크론탭이랑 비교하면 진짜 말도 안 되게 쉬워요. 크론탭은 문법 틀리면 아무것도 안 되는데, 코워크는 그냥 말로 하면 돼요."

## 실습 2 🔌 Enterprise search 플러그인 설치하기

### 📌 상황 설명

준혁은 지메일, 구글 드라이브, 노션을 모두 업무에 쓴다.  
장애 관련 메일, 구글 드라이브의 인프라 구성 문서, 노션의 장애 이력 노트가  
각기 다른 곳에 흩어져 있다.  
**Enterprise search 플러그인**을 설치하면 클로드가 세 곳을 한 번에 검색해준다.

### 🔧 설치 방법

**01단계** 왼쪽 패널에서 **[사용자 지정(Customize) → 플러그인 탐색]**을 누릅니다.  
검색창에 `Enterprise`를 입력해 플러그인을 찾습니다.  
찾았다면 **[+] 버튼**을 눌러 플러그인을 설치합니다.

> ℹ️ 이전에 플러그인을 설치한 적이 있다면 [플러그인 탐색]이 안 보일 수 있습니다.  
> 이때는 **[개인 플러그인]** 옆에 있는 **[+] 버튼**을 누르면 [플러그인 탐색]이 보입니다.

**02단계** 설치가 완료되면 왼쪽 패널 **[개인 플러그인]** 목록에  
`Enterprise search`가 표시됩니다.

### 📌 Enterprise search 플러그인의 두 가지 스킬

| 스킬 | 용도 | 준혁의 활용 예시 |
|------|------|-----------------|
| **digest** | 긴 문서나 이메일을 요약할 때 | 지난 24시간 장애 관련 메일·문서 요약 |
| **search** | 연결된 서비스에서 특정 정보를 찾을 때 | "광화문 사옥 스위치 관련 문서 찾아줘" |

> ℹ️ 이 플러그인을 사용하려면 **04장**에서 연결한 지메일, 구글 드라이브, 노션 커넥터가  
> 미리 연결되어 있어야 합니다.

### 💡 준혁의 한마디
> "지메일, 드라이브, 노션 따로 들어가서 검색하던 걸 이제 한 번에 해결해요. 특히 급할 때 '어제 광화문 장애 관련 메일 찾아줘' 하면 바로 뽑아줘서 진짜 편해요."

## 실습 3 📬 매일 종합 업무 리포트 자동으로 받기

### 📌 상황 설명

준혁은 매일 아침 출근하면 지난밤 상황을 파악하는 데 30분을 쓴다.  
이제 이 작업을 코워크 스케줄링으로 자동화해, 출근 전에 이미 리포트가 지메일로 와 있게 만들어보자.

### 🔧 스케줄링 설정 방법

**01단계** 코워크에서 **[New task(새 작업)]**를 시작하고  
**[+] 버튼**을 눌러 플러그인을 선택합니다.

**02단계** **[플러그인 → Enterprise search → digest]**를 순서대로 누릅니다.

**03단계** 아래 프롬프트를 입력해 스케줄링 설정을 클로드에게 지시합니다.

### 💬 프롬프트 예시

```
/digest 아침 8시마다 지난 24시간 인프라 관련 업무 요약 리포트를 내 지메일로 보내줘.
장애 관련 메일, 구글 드라이브 인프라 문서 업데이트, 노션 장애 이력 내용을 포함해줘.
```

**04단계** 작업 예약을 확인하는 화면이 나오면 **[일정]**을 눌러 승인합니다.

### ✅ 예상 결과 — 클로드 응답

```
AI
설정 완료됐어요!

매일 아침 8시 3분에 자동으로 실행되어
지난 24시간 활동을 ***@kt.com으로 발송합니다.

다이제스트에 포함되는 내용은 다음과 같아요:

• Gmail — 장애 관련 수신 메일 및 미회신 스레드
• Google Drive — 수정되거나 공유된 인프라 구성 문서
• Notion — 업데이트된 장애 이력 페이지 및 댓글

사이드바의 "Scheduled" 섹션에서 언제든지 일정을 확인하거나 수정할 수 있어요.
```

> ℹ️ 반복 작업은 서버 부하를 분산하기 위해 몇 분 정도의 지연이 자동으로 붙습니다.  
> 그래서 정각 8:00이 아니라 8:03으로 잡힐 수 있습니다.

## 실습 4 📋 스케줄 관리하기

### 📌 스케줄 확인 방법

왼쪽 패널에서 **[예정됨(Scheduled)]**을 눌러 스케줄링 관리 화면을 확인합니다.  
방금 요청했던 스케줄이 실제로 생성된 것을 볼 수 있습니다.

```
Scheduled tasks

• Daily infra digest
  매일 오전 8시, 지난 24시간 인프라 활동을 요약해 Gmail 초안으로 저장
  ⏰ 매일 오전 8:00
```

### 📌 스케줄 즉시 실행하기

스케줄이 실제로 작동하는지 바로 확인하고 싶다면 즉시 실행할 수 있습니다.

1. **[Scheduled]** 화면에서 확인하고 싶은 스케줄을 클릭합니다.
2. **[지금 실행]** 버튼을 누릅니다.
3. 실제 리포트가 생성되어 지메일로 발송됩니다.

### 📌 스케줄 수정·비활성화·삭제

스케줄 관리 화면에서 아래 작업을 할 수 있습니다.

| 작업 | 방법 |
|------|------|
| 수정 | 스케줄 클릭 → 내용 변경 후 저장 |
| 일시 비활성화 | 스케줄 옆 토글 끄기 |
| 삭제 | 스케줄 클릭 → 삭제 버튼 |
| 말로 수정 | 코워크에서 "인프라 리포트 시간을 오전 7시 30분으로 바꿔줘" |

> ℹ️ 스케줄을 관리하는 가장 간단한 방법은 그냥 코워크에서 클로드에게 말로 지시하는 겁니다.

### 💡 준혁의 한마디
> "스케줄 수정도 그냥 말로 해요. '리포트 시간 8시에서 7시 반으로 바꿔줘' 하면 바로 바꿔줘요. 크론탭 수정하면서 문법 오류 잡던 시절이 그립지가 않네요."
